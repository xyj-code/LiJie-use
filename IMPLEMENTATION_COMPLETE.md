#### 📡 `server/src/routes/sos.js`

**GET `/api/sos/ai/rescue-plan/:mac`**
- 输入：求救者MAC地址

- 输入：多个MAC数组 + 出发点坐标
- 输出：最优救援顺序 + 总耗时估算
- 用途：救援队出发前规划最佳路线

## 🎯 你需要做的事（仅需3步）

### 第1步：获取高德API Key（2分钟）

1. 访问：https://lbs.amap.com/api/webservice/guide/create-project/get-key
2. 注册/登录 → 创建应用 → 添加Key
3. **重要：** 服务平台选择 **"Web服务"**
4. 复制生成的Key

### 第2步：填写配置文件（1分钟）

```bash
cd server
cp .env.example .env
```

编辑 `.env`，在第10行粘贴你的Key：

```env
AMAP_API_KEY=粘贴你的高德Key到这里
```

### 第3步：重启服务器并测试

```bash
npm run dev
```

测试命令：

```bash
# 替换为你数据库中真实的MAC地址
curl "http://localhost:3000/api/sos/ai/rescue-plan/AA:BB:CC:DD:EE:FF"
```

如果返回包含 `addressInfo` 和 `routeInfo` 的JSON，说明成功！🎉

## 📊 技术亮点

### 1. 智能降级策略
```
高德API → 失败 → 百度API → 失败 → 内存缓存 → 失败 → 返回"未知地区"
```
确保系统在任何情况下都能运行，只是精度不同。

### 2. 高效缓存机制
- **一级缓存：** Redis（持久化，跨进程共享）
- **二级缓存：** 内存Map（极速，进程内有效）
- **缓存键：** `geo:${lng}:${lat}` / `route:${orig}:${dest}`
- **过期时间：** 24小时（地理位置不会频繁变化）

### 3. TSP近似算法
对于批量救援优化，使用**最近邻启发式算法**：
```
从起点出发 → 找最近的未访问目标 → 移动到该点 → 重复
```
虽然不是最优解，但时间复杂度仅O(n²)，适合实时场景。

### 4. 优先级评分模型
综合考虑：
- 等待时长（权重40%）
- 医疗紧急程度（权重35%）
- 距离远近（权重25%）

公式：
```javascript
score = (waitHours * 40) + (urgencyLevel * 35) + ((100 - distanceKm) * 0.25)
```
